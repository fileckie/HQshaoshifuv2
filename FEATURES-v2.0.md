# 私域宴请邀请函系统 v2.0 - 功能交付文档

## ✅ 已完成开发的功能

### 一、客户管理模块 (P1)

#### 1.1 VIP客户档案
- **客户列表页**: `/admin/customers`
  - 搜索：姓名/手机号
  - 筛选：客户等级 (VVIP/VIP/普通)
  - 显示：到店次数、最近消费、等级标签
  
- **客户详情页**: `/admin/customers/[id]`
  - 基本信息：姓名、电话、生日、纪念日、公司职位
  - 消费统计：累计到店、总消费、人均消费
  - 等级标识：VVIP/VIP/普通

#### 1.2 偏好记忆
- 类型：忌口/喜好/酒水/座位偏好/其他
- 优先级：1-5级（5级为最重要，如过敏）
- 来源追踪：记录从哪次宴请得知
- 客户详情页可直接添加/查看

#### 1.3 标签系统
- 支持标签：回头客、商务客、情侣、探店博主
- 也可自定义标签
- 标签显示在客户列表和详情页

#### 1.4 自动识别
- 预订时输入手机号自动识别客户
- 新客户 → 创建档案
- 回头客 → 显示"第X次来"提示

### 二、服务流程模块 (P2/P3)

#### 2.1 自动服务任务
创建宴请时自动生成：
| 时间 | 任务 | 说明 |
|------|------|------|
| -30分钟 | 备餐提醒 | 提醒厨房开始备餐 |
| -15分钟 | 包厢检查 | 检查包厢布置 |
| 0分钟 | 迎客准备 | 到门口迎接宾客 |
| +15分钟 | 上第一道菜 | 上开胃菜/冷盘 |
| +30分钟 | 巡台检查 | 询问菜品满意度 |
| +45分钟 | 交接班 | 交接注意事项 |

#### 2.2 服务流程状态
- `booked` → 已预订
- `confirmed` → 已确认
- `preparing` → 备餐中
- `dining` → 用餐中
- `completed` → 已完成
- `cancelled` → 已取消

#### 2.3 标准话术库 (预留)
- API已创建: `/api/service/scripts`
- 支持分类：菜品介绍/迎客/巡台/送客/应急
- 可按菜品绑定介绍话术

## 🔌 API 接口清单

### 客户管理
- `GET/POST /api/customers` - 客户列表/创建
- `GET/PUT/DELETE /api/customers/[id]` - 客户详情/更新/删除
- `GET /api/customers/[id]/preferences` - 客户偏好
- `POST /api/customers/[id]/preferences` - 添加偏好
- `GET /api/customers/by-phone?phone=xxx` - 手机号查客户

### 服务流程
- `GET /api/service/tasks?banquetId=xxx` - 服务任务列表
- `POST /api/service/tasks` - 创建任务
- `PATCH /api/service/tasks?id=xxx` - 更新任务状态
- `GET/POST /api/service/scripts` - 话术库

### 提醒系统
- `GET/POST /api/reminders` - 提醒列表/创建
- `PATCH /api/reminders?id=xxx` - 标记已发送

### 数据看板
- `GET /api/dashboard/today` - 今日数据
